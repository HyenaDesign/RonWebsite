Ron online. I am an AI companion, not a doctor or lawyer.

Ron is lightweight by design: Electron, React, TypeScript, a tiny desktop surface, and streamed OpenAI Chat Completions through the main process. It keeps only a short recent-turn memory, because a desk buddy does not need to pretend it is your whole life.

API key privacy

Your key, your device.

Ron does not ship with an API key. Every user must provide their own OpenAI API key on first launch. The key is saved locally in Electron's app data folder and should stay out of GitHub, packages, and public places where secrets go to have a bad afternoon.

First launch setup

  1. Launch Ron.
  2. Open settings with the gear icon.
  3. Paste your OpenAI API key.
  4. Click the check button.

Never commit API keys, .env files, packaged secrets, or local app data.

Safety notes

Ron is a companion, not a professional.

Ron is not a doctor or lawyer. It should not provide medical advice, legal advice, diagnoses, treatment instructions, medication dosing, legal strategy, or interpretations of rights and obligations.

For medical or legal concerns, contact qualified professionals or official sources. Ron can be useful company, but it should not be the authority in high-stakes moments.

Unsigned Windows build

SmartScreen may warn you.

Ron is distributed without an EV code-signing certificate. Windows SmartScreen may show a blue warning screen when the app is opened.

Only run unsigned apps from sources you trust. If Ron arrives from an unexpected source, do not run it.

If you trust the source

  1. Click More info.
  2. Confirm the publisher is unknown or unsigned.
  3. Click Run anyway.

Developer notes

Build Ron locally.

Ron requires Windows 10 or newer, Node.js 20+, npm, and an OpenAI API key for actual use.

Build npm run build
